# pm-workbench v1.0.5 release notes

## What this release means

`pm-workbench` is moving from an early GitHub-packaged skill into a more complete 1.0 product skill.

The core shift is not just “more files.”
It is that the repo now has:

- a clearer product thesis
- stronger PM judgment positioning
- reusable artifact coverage across the main workflow set
- benchmark assets that help compare it against generic AI
- better product-leader / founder support
- lightweight local validation so trust does not rely only on README prose

## Highlights since the older GitHub-facing baseline

### 1. Stronger PM judgment layer

- clearer routing across 9 core PM workflows
- stronger “critical premise first” behavior in high-frequency scenarios
- more explicit recommendation, trade-off, and non-decision guidance

### 2. Artifact-first output system

- expanded reusable PM artifact templates across the workflow set
- better support for outputs that can survive meetings, leadership syncs, and planning review

### 3. Benchmark and proof layer

- benchmark kit with scenarios, rubric, and scorecard
- three worked benchmark comparisons
- README-ready benchmark visual summary
- benchmark contribution guidance for future evidence-building

### 4. Product-leader and founder support

- product leader playbook
- stronger prioritization, portfolio review, and operating review coverage
- founder strategy and founder business review examples

### 5. Better repo maturity

- clearer README and onboarding flow
- richer examples index
- roadmap and changelog continuity
- local validation script to catch structural drift early

## Who this is for

`pm-workbench` is for:

- PMs who need sharper framing, evaluation, prioritization, and communication
- product leaders who need clearer portfolio and leadership-ready artifacts
- founders who need structured trade-off support instead of generic AI advice

## What to expect next

1.0 is a strong publishing point, not the end state.
Likely next areas after release:

- more benchmark coverage
- founder / business review depth
- sharper release / distribution packaging
- continued refinement of product-leader artifacts

## Bottom line

This release is the point where `pm-workbench` starts to feel like a real productized PM skill:
not just interesting, but easier to try, easier to trust, and easier to use in real decision work.
